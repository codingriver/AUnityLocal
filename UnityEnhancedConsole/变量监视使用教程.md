# 变量监视 (Watch) 使用教程

## 1. 简介

变量监视是 UnityEnhancedConsole 的内置功能，允许你在编辑器中**实时追踪任意变量的值**，无需反复编写 `Debug.Log`。

核心优势：
- **零发布开销** — 所有 API 标记 `[Conditional("UNITY_EDITOR")]`，正式构建中自动移除，不产生任何运行时代价。
- **分组管理** — 用 `/` 路径语法将变量归组，一目了然。
- **历史记录** — 自动记录值变化历程，可随时回溯。
- **图表可视化** — 对数值、向量类型实时绘制折线图。
- **导出 & 持久化** — 支持 CSV/JSON 导出，可选文件持久化跨域保留数据。

---

## 2. 快速开始

### 2.1 打开变量监视面板

1. 菜单栏打开 **Enhanced Console** 窗口。
2. 点击窗口顶部的 **Watch** 标签页，切换到变量监视面板。

### 2.2 最简示例

```csharp
using UnityEngine;
using UnityEnhancedConsole;

public class PlayerController : MonoBehaviour
{
    public float health = 100f;
    public Vector3 velocity;

    void Update()
    {
        // 一行代码即可监视
        Watch.Set("Player/Health", health);
        Watch.Set("Player/Velocity", velocity);
    }
}
```

运行后，Watch 面板将实时显示 `Player` 分组下的 `Health` 和 `Velocity`。

## 3. API 详解

### 3.1 手动设置值 — `Watch.Set`

最常用的方式。每帧（或在值变化时）调用，面板即时刷新。

```csharp
// 浮点数（默认 F2 格式）
Watch.Set("FPS", 1f / Time.deltaTime);

// 整数
Watch.Set("Score", currentScore);

// 布尔值（面板显示彩色圆点 ● / ○）
Watch.Set("IsGrounded", controller.isGrounded);

// 向量（默认 F2 格式，面板显示各分量）
Watch.Set("Camera/Position", Camera.main.transform.position);

// 颜色（面板显示色块预览）
Watch.Set("Ambient", RenderSettings.ambientLight);

// 字符串
Watch.Set("State", currentState.ToString());

// 任意对象（调用 ToString() 显示）
Watch.Set("ActiveWeapon", weapon);
```

**自定义格式**：所有数值类型重载都支持可选的 `format` 参数：

```csharp
Watch.Set("Precise/Position", transform.position, "F4");  // 4 位小数
Watch.Set("Hex/Color", color, "X8");
```

### 3.2 自动监视 — `Watch.Auto`

注册一次，自动在每个编辑器 Update 周期求值，无需手动每帧调用。

```csharp
void OnEnable()
{
    // 参数: 名称, 取值函数, 所属对象(用于自动清理), 格式(可选)
    Watch.Auto("Enemy/Count",
        () => FindObjectsOfType<Enemy>().Length,
        this);

    Watch.Auto("Memory/Allocated",
        () => (System.GC.GetTotalMemory(false) / 1048576f).ToString("F1") + " MB",
        this);
}
```

> **自动清理**：当 `owner` 对象被销毁后，该监视条目会在下一次 Update 时自动移除，无需手动管理。

### 3.3 移除

```csharp
// 移除单个条目
Watch.Remove("Player/Health");

// 移除某个对象注册的所有条目（常用于 OnDisable/OnDestroy）
Watch.RemoveAll(this);

// 清空全部
Watch.Clear();
```

### 3.4 暂停 / 恢复

暂停后条目停止更新，但保留在面板中，方便冻结某一瞬间的值进行对比。

```csharp
Watch.SetPaused("Player/Health", true);   // 暂停
Watch.SetPaused("Player/Health", false);  // 恢复
```

## 4. 分组与路径

使用 `/` 分隔符将变量归入同一组，面板以可折叠的组头显示：

```csharp
Watch.Set("Physics/Gravity",    Physics.gravity.y);
Watch.Set("Physics/FixedDelta", Time.fixedDeltaTime);
Watch.Set("Physics/Solver",     Physics.defaultSolverIterations);

Watch.Set("AI/Agent/Speed",  agent.speed);
Watch.Set("AI/Agent/Target", agent.destination);
```

面板呈现：

```
▼ Physics
    Gravity        -9.81
    FixedDelta     0.02
    Solver         6
▼ AI/Agent
    Speed          3.50
    Target         (10.00, 0.00, 5.00)
```

> 取最后一个 `/` 进行分割。`"AI/Agent/Speed"` → 组 `AI/Agent`，显示名 `Speed`。

## 5. 面板功能

### 5.1 工具栏

| 按钮 | 功能 |
|------|------|
| **Clear** | 清空所有监视条目 |
| **Pause All** | 暂停全部条目更新 |
| **Resume All** | 恢复全部条目更新 |
| **搜索框** | 按名称实时过滤条目 |
| **Export** | 导出当前数据为 CSV 或 JSON |
| **⚙ Settings** | 打开偏好设置窗口 |

### 5.2 监视列表

- **Name** — 变量显示名（含组路径折叠）
- **Value** — 当前值，带有类型特定的视觉增强：
  - 布尔：绿色 `●` (true) / 灰色 `○` (false)
  - 颜色：值前方显示颜色方块
  - 数值：显示趋势箭头 `▲` (上升) / `▼` (下降) / `─` (不变)
- **Changes** — 值变化总次数
- **Last** — 距上次变化的时间

**值变化闪烁**：当值发生改变时，该行背景短暂高亮（默认 0.3 秒），便于在大量条目中快速捕捉变化。

### 5.3 右键菜单

在任意条目上右键：

| 选项 | 说明 |
|------|------|
| Copy Value | 复制当前值到剪贴板 |
| Copy Name | 复制完整变量名到剪贴板 |
| Pause / Resume | 暂停或恢复该条目 |
| Remove | 移除该条目 |
| Clear All | 清空全部 |

**快捷键**：`Ctrl+C`（macOS `Cmd+C`）快速复制选中条目的 `名称: 值`。

### 5.4 历史面板

选中任意条目后，下方显示该变量的历史变化记录（最新在上）：

```
[12:34:56.789] Frame 1024  →  98.50
[12:34:56.123] Frame 1020  →  100.00
```

### 5.5 图表视图

对 **数值 (Float/Integer)** 和 **向量 (Vector)** 类型，历史面板支持切换为实时折线图：

1. 选中一个数值/向量类型的条目。
2. 点击历史面板右上角的 **📊 Graph** 按钮，切换到图表视图。
3. 再次点击 **📋 List** 切换回列表视图。

图表特性：
- **自动缩放** — Y 轴根据数据范围自适应。
- **向量分量** — Vector2/Vector3 分别以红(X)、绿(Y)、蓝(Z) 绘制。
- **鼠标悬停** — 显示该时刻的精确值。
- **零线** — 数据跨零时显示虚线零参考线。

## 7. 偏好设置

点击工具栏 **⚙** 按钮打开设置窗口：

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| History Depth | 300 | 每个条目保留的历史记录数（1 ~ 10000） |
| Max Entries | 1000 | 同时监视的最大条目数（1 ~ 100000） |
| Auto Update Interval | 0s | 自动监视的求值间隔，0 = 每帧 |
| Flash Duration | 0.3s | 值变化时行高亮持续时间（0 ~ 5s） |
| Graph Time Range | 5s | 图表显示的时间窗口（1 ~ 300s） |
| Show Trend Arrow | ✅ | 是否显示趋势箭头 |
| Persist to File | ❌ | 是否将监视数据持久化到文件（跨域重载保留） |

## 8. 实用场景

### 场景一：调试角色控制器

```csharp
public class CharacterDebug : MonoBehaviour
{
    CharacterController _cc;

    void Awake()
    {
        _cc = GetComponent<CharacterController>();
    }

    void Update()
    {
        Watch.Set("Character/IsGrounded", _cc.isGrounded);
        Watch.Set("Character/Velocity",   _cc.velocity);
        Watch.Set("Character/Speed",      _cc.velocity.magnitude);
    }

    void OnDestroy()
    {
        Watch.RemoveAll(this); // 可选：对象销毁时主动清理
    }
}
```

### 场景二：性能监控

```csharp
public class PerfMonitor : MonoBehaviour
{
    void OnEnable()
    {
        Watch.Auto("Perf/FPS",
            () => Mathf.RoundToInt(1f / Time.unscaledDeltaTime),
            this);

        Watch.Auto("Perf/DrawCalls",
            () => UnityStats.drawCalls,
            this);

        Watch.Auto("Perf/Triangles",
            () => $"{UnityStats.triangles / 1000}K",
            this);
    }
}
```

### 场景三：AI 状态追踪

```csharp
public class AIDebug : MonoBehaviour
{
    FSM _fsm;
    NavMeshAgent _agent;

    void Update()
    {
        Watch.Set("AI/State",           _fsm.CurrentState.Name);
        Watch.Set("AI/Target",          _agent.destination);
        Watch.Set("AI/RemainingDist",   _agent.remainingDistance);
        Watch.Set("AI/HasPath",         _agent.hasPath);
    }
}
```

### 场景四：网络同步调试

```csharp
public class NetDebug : MonoBehaviour
{
    void Update()
    {
        Watch.Set("Net/RTT",         NetworkManager.Singleton.NetworkConfig.NetworkTransport.GetCurrentRtt(0));
        Watch.Set("Net/PacketLoss",  GetPacketLoss(), "P1");  // 百分比格式
        Watch.Set("Net/Connected",   NetworkManager.Singleton.IsConnectedClient);
    }
}
```

### 场景五：动画参数监控

```csharp
public class AnimDebug : MonoBehaviour
{
    Animator _anim;

    void Update()
    {
        Watch.Set("Anim/Speed",      _anim.GetFloat("Speed"));
        Watch.Set("Anim/IsAttack",   _anim.GetBool("IsAttacking"));
        Watch.Set("Anim/StateInfo",  _anim.GetCurrentAnimatorStateInfo(0).shortNameHash);
    }
}
```

## 9. 最佳实践

### 命名规范

```csharp
// ✅ 推荐：使用分组路径，语义清晰
Watch.Set("Player/Health", hp);
Watch.Set("Player/Mana",   mp);

// ❌ 避免：扁平命名，条目多时难以管理
Watch.Set("hp", hp);
Watch.Set("mp", mp);
```

### 避免在高频循环中创建临时字符串

```csharp
// ✅ 使用类型化重载，内部格式化
Watch.Set("Value", 3.14f);

// ❌ 手动 ToString 产生 GC
Watch.Set("Value", 3.14f.ToString("F2"));
```

### 及时清理

```csharp
void OnDisable()
{
    // Auto 模式会自动清理（基于 WeakReference）
    // 手动 Set 模式建议在不需要时主动移除
    Watch.RemoveAll(this);
}
```

### 善用暂停

在排查瞬间值变化时，先暂停目标条目，仔细检查历史记录和图表，确认后再恢复。

```csharp
// 也可以通过面板右键菜单 Pause/Resume，无需写代码
Watch.SetPaused("Player/Health", true);
```

## 10. 注意事项

1. **仅编辑器可用** — 所有 `Watch.*` 调用在非编辑器构建中被编译器完全移除（`[Conditional("UNITY_EDITOR")]`），不会影响发布包体积和性能。

2. **主线程调用** — `Watch.Set` / `Watch.Auto` 应在主线程调用。若需从后台线程监视数据，请通过主线程中转。

3. **历史仅记录变化** — 相邻两次调用如果值相同，不会产生新的历史记录。如需每帧记录，确保值有变化或附加帧号。

4. **Auto 模式的生命周期** — `owner` 参数使用 `WeakReference` 追踪。当 `owner` 被 GC 回收或 Destroy 后，自动监视条目将在下一个 Editor Update 被清除。

5. **数据不跨 Play Mode** — 默认情况下退出 Play Mode 时所有监视数据清空。如需保留，请在设置中开启 **Persist to File**。

## 11. 常见问题

**Q: 面板没有显示我的监视变量？**
> 确认当前处于 **Watch** 标签页（非 Log 标签页）。检查搜索框是否有过滤文本。确认代码中的 `Watch.Set` 调用确实被执行到。

**Q: 图表按钮没有出现？**
> 图表仅对 Float、Integer、Vector 类型且有 2 条以上历史记录时可用。布尔/字符串/颜色类型不支持图表。

**Q: Watch.Auto 注册后没有更新？**
> 检查 `owner` 对象是否已经被销毁。如果 `owner` 为 null 或已销毁，自动监视会被清除。

**Q: 导出文件在哪里？**
> 点击 Export 后会弹出系统文件保存对话框，自行选择保存路径。

**Q: 历史记录数量不够？**
> 打开设置（⚙ 按钮），调大 **History Depth**（最大 10000）。
